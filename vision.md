## ١. الفكرة الجوهرية

تطبيق ويب يعمل كـ **بيئة عمل متكاملة** يُحوَّل فيها الملف إلى مشروع حي — والمحادثة مع الذكاء الاصطناعي هي أداة التحكم الرئيسية في كل شيء.

**الغاية:** تمكين أي شخص من رفع ملف بيانات والحصول على تحليل، نتائج، وملفات جديدة — كل ذلك من خلال المحادثة، دون خبرة تقنية.

---

## ٢. طبيعة التطبيق

| المحور | التفصيل |
|---|---|
| **اسم التطبيق** | **DataChat** |
| **الأساس** | واجهة محادثة ذكية (Chat-First) |
| **البيئة** | مشروع متكامل يضم الملف + المحادثة + النتائج + الملفات المولّدة |
| **اللغة** | عربي بالكامل، RTL |
| **الذكاء الاصطناعي** | Google Gemini — يرى بيانات الملف حصراً وينفذ أوامر إنشاء الملفات |
| **دعم الجوال** | تصميم متجاوب يعمل على الحاسب والجوال والتابلت |

## ٤. آلية عمل التطبيق

### ٤.١ رفع الملف وإنشاء المشروع
- المستخدم يرفع ملف Excel أو CSV أو PDF أو Word
- يُنشأ "مشروع" جديد يحتوي على: الملف + محادثة فارغة
- الملف يظهر فوراً في لوحة الملفات على اليسار
- **يُحفَظ الملف بالكامل في قاعدة البيانات دون أي حد للصفوف** — لضمان دقة النتائج الكاملة
- يرسل الذكاء الاصطناعي رسالة ترحيب بسيطة تُعلم المستخدم بنجاح الرفع وتدعوه لطرح أسئلته

### ٤.٢ التحليل عند الطلب
- **لا يوجد تحليل تلقائي** — الذكاء الاصطناعي ينتظر توجيه المستخدم
- المستخدم يطلب ما يحتاجه متى أراد:
  - "حلّل الملف وأخبرني بأبرز الملاحظات"
  - "كم عدد الصفوف والأعمدة؟"
  - "ما أنواع البيانات الموجودة؟"
- الذكاء الاصطناعي يستجيب بناءً على الطلب فقط

### ٤.٣ المحادثة الحرة
- المستخدم يكتب ما يريد بلغته الطبيعية
- الذكاء الاصطناعي مُقيَّد ببيانات الملف حصراً
- النتائج تظهر في المحادثة: نصوص، جداول، أرقام

### ٤.٤ توليد الملفات من المحادثة
المستخدم يطلب بكلامه العادي، الذكاء الاصطناعي ينفذ ويُعيد الملف مباشرةً في الشات:

| الطلب | الناتج في المحادثة |
|---|---|
| "أنشئ Excel بأسماء الأعضاء والمدن فقط" | 📥 أعضاء_المدن.xlsx |
| "اعمل تقرير PDF بمجاميع كل تخصص" | 📥 تقرير_التخصصات.pdf |
| "أعطني الأعضاء الذين انضموا بعد 2020 في ملف" | 📥 أعضاء_2020.xlsx |
| "ضع ملخص التحليل في ملف Word" | 📥 ملخص_التحليل.docx |

- كل ملف مُولَّد يُضاف تلقائياً إلى لوحة الملفات في المشروع
- يمكن تنزيله من بطاقة التنزيل في المحادثة أو من لوحة الملفات

### ٤.٥ لوحة الملفات (File Panel)
نافذة جانبية تعرض كل ما في المشروع:
- **الملف الأصلي:** أيقونة + اسم + تاريخ الرفع
- **الملفات المولّدة:** كل ملف أنشأه الذكاء الاصطناعي مع نوعه وتاريخه
- **النتائج المحفوظة:** ردود مهمة حفظها المستخدم
- النقر على أي ملف → تنزيل فوري

## ٥. الميزات الإضافية

### أ. المحادثة الحرة الكاملة (بدون أوامر سريعة)
واجهة المحادثة نظيفة تماماً — **لا توجد أزرار أوامر سريعة**. المستخدم يكتب ما يريده بحرية كاملة والذكاء الاصطناعي يستجيب. هذا يُبقي الواجهة بسيطة ويمنح المستخدم حرية التعبير دون قيود.

### ب. الذاكرة السياقية
الذكاء الاصطناعي يتذكر كل المحادثة ويبني عليها:
> "عد إلى النتيجة التي أعطيتني إياها قبل قليل وضعها في PDF"
> "استخدم نفس الفلتر السابق لكن على عمود آخر"

### ج. الجداول التفاعلية في المحادثة
عندما يُعيد الذكاء الاصطناعي جدولاً → يظهر كجدول **تفاعلي كامل** داخل الشات:
- ترتيب الأعمدة تصاعدياً وتنازلياً بالضغط على العنوان
- تصفية البيانات مباشرة داخل الجدول
- البحث داخل نتائج الجدول
- تنزيل الجدول مباشرة كملف Excel من داخل المحادثة

### د. دعم ملفات متعددة (مستقبلاً)
رفع أكثر من ملف في نفس المشروع وربطها:
> "قارن بين ملف هذا العام وملف العام الماضي"

### هـ. تصدير المشروع كاملاً
تنزيل كل محتوى المشروع (الملف الأصلي + كل الملفات المولّدة) في ملف ZIP واحد

### ح. معالجة الأخطاء

| الموقف | السلوك المُعتمد |
|---|---|
| **الذكاء الاصطناعي لم يفهم الطلب** | يطلب توضيحاً من المستخدم بأسلوب ودي |
| **الملف تالف أو غير مدعوم** | رسالة خطأ واضحة تشرح السبب + دعوة لرفع ملف صحيح |
| **انقطاع الاتصال بالـ API** | إعادة المحاولة تلقائياً مرتين، ثم إشعار للمستخدم مع زر "أعد المحاولة" |
| **الملف يتجاوز الحد الأقصى** | رفض الرفع مع رسالة توضح السبب والحد المسموح |

**مبدأ عام:** كل رسالة خطأ في DataChat يجب أن تكون:
- واضحة وبسيطة (لا رموز تقنية)
- تخبر المستخدم ماذا حدث
- تقترح الخطوة التالية

### ط. حذف المشاريع

- **من يحذف:** كل مستخدم يستطيع حذف مشاريعه الخاصة فقط
- **التأكيد:** تظهر نافذة تأكيد قبل الحذف النهائي تحذّر من أن العملية لا يمكن التراجع عنها
- **ما يُحذَف:** المشروع + الملف الأصلي + كل الملفات المولّدة + كامل المحادثة

```
┌──────────────────────────────────────────┐
│  ⚠️  تأكيد الحذف                         │
│                                          │
│  هل تريد حذف مشروع "تقرير المبيعات"؟   │
│                                          │
│  سيُحذَف نهائياً:                        │
│  • الملف الأصلي                          │
│  • 3 ملفات مُولَّدة                      │
│  • كامل المحادثة (12 رسالة)             │
│                                          │
│  ⚠️ لا يمكن التراجع عن هذا الإجراء      │
│                                          │
│  [إلغاء]              [نعم، احذف نهائياً] │
└──────────────────────────────────────────┘
```

### ي. البحث في المحادثات السابقة

خاصية بحث شاملة تتيح للمستخدم إيجاد أي معلومة في تاريخ محادثاته:

```
┌─────────────────────────────────────────────────────┐
│  🔍 ابحث في محادثاتك...                             │
├─────────────────────────────────────────────────────┤
│  نتائج البحث عن "مجاميع المدن"                      │
│  ─────────────────────────────────────────           │
│  📊 تقرير المبيعات  •  15/6/2026                    │
│  "إليك مجاميع الأعضاء حسب المدينة: الرياض 245..."   │
│                                           [افتح ▶]  │
│  ─────────────────────────────────────────           │
│  📋 بيانات الموظفين  •  10/6/2026                   │
│  "مجاميع الموظفين حسب المدينة تُظهر أن..."           │
│                                           [افتح ▶]  │
└─────────────────────────────────────────────────────┘
```

- البحث يشمل: رسائل المستخدم + ردود الذكاء الاصطناعي + أسماء الملفات
- النقر على نتيجة يفتح المشروع مباشرةً عند الرسالة المعنية

### ز. تاب إعدادات الذكاء الاصطناعي (AI Settings)
صفحة مستقلة في التطبيق تتيح التحكم الكامل في سلوك الذكاء الاصطناعي وإعداداته:

#### ١. إدارة الأوامر النصية (System Prompts)
- عرض الأوامر الحالية المُعطاة للذكاء الاصطناعي مع إمكانية تعديلها مباشرة
- حفظ نسخ متعددة من الأوامر حسب الاستخدام:

| الوضع | مثال |
|---|---|
| تحليل مالي | "ركّز على الأرقام والمجاميع والنسب المئوية..." |
| موارد بشرية | "ركّز على بيانات الموظفين والحضور والأداء..." |
| مخصص | يكتبه المستخدم بنفسه |

- زر "إعادة الضبط" للرجوع للإعدادات الافتراضية

#### ٢. إدارة API والنموذج
- حقل إدخال الـ API Key مع إخفاء القيمة (••••••)
- زر **"تحقق من الاتصال"** يختبر الـ API فوراً ويعرض النتيجة ✅ أو ❌
- قائمة اختيار النموذج: (Gemini 1.5 Pro / Gemini 1.5 Flash / إلخ)
- عرض حالة الاتصال الحالية في أعلى الصفحة

#### ٣. إعدادات سلوك الذكاء الاصطناعي
- **درجة الإبداع (Temperature):** شريط تمرير من 0 (دقيق وحرفي) إلى 1 (مبدع ومتنوع)
- **الحد الأقصى للرد (Max Tokens):** التحكم في طول الإجابة
- **لغة الرد الافتراضية:** عربي / إنجليزي / تلقائي

#### ٤. إعدادات الملفات
- الحد الأقصى لحجم الملف المرفوع
- عدد الصفوف المرسلة للذكاء الاصطناعي
- تنسيقات الملفات المسموح بها (Excel، CSV، PDF، Word)

#### ٥. إعدادات الواجهة
- الوضع الداكن / الفاتح (Dark / Light Mode)
- حجم الخط في المحادثة
- إظهار / إخفاء الأوامر السريعة

#### ٦. سجل الاستخدام (للمعلومات فقط)
- عدد الرسائل المُرسلة للذكاء الاصطناعي
- عدد الملفات المُولَّدة
- تاريخ آخر استخدام للتطبيق

### و. مؤشرات التقدم الحي (Live Progress Indicators)
واجهة المحادثة تعرض ما يفعله الذكاء الاصطناعي لحظةً بلحظة — بدلاً من شاشة انتظار صامتة:

| الحالة | ما يظهر في الشات |
|---|---|
| بدء المعالجة | 🔄 جاري قراءة الملف... |
| تحليل البيانات | 📊 جاري تحليل البيانات... |
| إنشاء ملف | 📝 جاري إنشاء ملف Excel... |
| اكتمال | ✅ تم — الملف جاهز للتنزيل |

**آلية التنفيذ التقنية — Streaming عبر SSE أو WebSocket:**
```
Backend يبدأ المعالجة
    ↓ يرسل: { status: "reading_file",  message: "جاري قراءة الملف..." }
    ↓ يرسل: { status: "analyzing",     message: "جاري تحليل البيانات..." }
    ↓ يرسل: { status: "generating",    message: "جاري إنشاء التقرير..." }
    ↓ يرسل: { status: "done",          result: "..." + رابط الملف }
الواجهة تعرض كل خطوة فور وصولها — بدون انتظار
```

**التقنية المستخدمة:** Server-Sent Events (SSE) أو WebSocket
**الفائدة:** تجربة مستخدم احترافية تُشعره بأن النظام يعمل وليس متوقفاً

### ٦. شعار التطبيق — أيقونة + نص

```
  ╔═══╗
  ║ ◈ ║  DataChat
  ╚═══╝
  أيقونة    نص
```

- **الأيقونة:** رمز يجمع بين المحادثة والبيانات (فقاعة حوار + نقاط بيانية)
- **النص:** "DataChat" بخط Cairo Bold باللون الرئيسي البنفسجي
- يظهر الشعار كاملاً (أيقونة + نص) في: شاشة تسجيل الدخول، شاشة الإعداد الأولي، الشريط العلوي للحاسب
- يظهر الأيقونة فقط في: Sidebar الحاسب المطوي، Bottom Nav الجوال

### ١٢. ثنائية اللغة — عربي / إنجليزي

```
الشريط العلوي:
┌─────────────────────────────────────────────────────┐
│  DataChat ◈          🔍          [AR | EN]  ☀️  👤  │
└─────────────────────────────────────────────────────┘
```

- زر **[AR | EN]** في الشريط العلوي لتبديل اللغة فوراً
- عند تحديد **العربية:** الواجهة RTL كاملة، جميع النصوص عربية
- عند تحديد **الإنجليزية:** الواجهة LTR كاملة، جميع النصوص إنجليزية
- يُحفَظ اختيار اللغة في إعدادات المستخدم ويبقى عبر الجلسات
- **محتوى المحادثة** يبقى كما أُدخِل بغض النظر عن لغة الواجهة

### ١٣. الإشعارات داخل التطبيق

**أنواع الإشعارات:**

```
                            │                         ✕   │
```

- يختفي تلقائياً بعد **4 ثوانٍ** أو عند الضغط على ✕
- لا توجد صفحة سجل إشعارات — تظهر لحظياً فقط

### الأدوار المتاحة

## ٦.أ الهوية البصرية للتطبيق

التطبيق يمتلك هوية بصرية متكاملة وموحّدة تنعكس على كل عناصره:

### مبادئ التصميم
- **الاتساق:** نفس الألوان والخطوط والمسافات في كل شاشة
- **الاحترافية:** تصميم يليق ببيئة العمل المؤسسي
- **البساطة:** واجهة نظيفة بلا فوضى بصرية
- **RTL أولاً:** كل عنصر مصمم من الأساس للعربية

### عناصر الهوية البصرية

| العنصر | التفصيل |
|---|---|
| **الشعار** | شعار التطبيق يظهر في شريط التنقل وصفحة تسجيل الدخول |
| **لوحة الألوان** | لون رئيسي + لون ثانوي + ألوان حالة (نجاح / خطأ / تحذير) |
| **الخط** | خط عربي احترافي موحّد في كل الواجهة |
| **الأيقونات** | مجموعة أيقونات متسقة لكل أنواع الملفات والأحداث |
| **البطاقات** | تصميم موحّد لبطاقات الملفات والرسائل والنتائج |
| **الأزرار** | أسلوب موحّد للأزرار الأساسية والثانوية |

### تطبيق الهوية على ملفات PDF المولّدة
كل ملف PDF ينشئه الذكاء الاصطناعي يحمل نفس الهوية البصرية:

```
┌──────────────────────────────────────────┐
│  [شعار التطبيق]          [اسم المشروع]  │  ← رأس الصفحة
│  ══════════════════════════════════════  │
│                                          │
│  عنوان التقرير                           │
│  التاريخ: 2026-06-15                     │
│                                          │
│  ┌────────────────────────────────────┐  │
│  │         [جدول البيانات]           │  │  ← محتوى منسق
│  └────────────────────────────────────┘  │
│                                          │
│  ══════════════════════════════════════  │
│  صفحة 1 من 3              [اسم المُعِد]  │  ← تذييل الصفحة
└──────────────────────────────────────────┘
```

**مكونات PDF الاحترافي:**
- رأس صفحة ثابت: شعار + اسم المشروع + التاريخ
- تذييل ثابت: رقم الصفحة + اسم المُعِد
- جداول منسقة بألوان الهوية البصرية
- خط عربي مدمج داخل الملف لضمان العرض الصحيح
- هوامش وتباعد احترافي

### ٧.٦ المفتاح الجوهري — Function Calling

هذا هو العمود الفقري للتطبيق. بدلاً من أن يكتب Gemini الإجابة نصاً فقط، يستدعي دالة محددة في الـ Backend تنفّذ العمل الفعلي:

```

         ↓
Gemini يقرر استدعاء: create_excel_file(filter="city=X", columns=["name"])
         ↓
Backend ينشئ الملف فعلياً بـ openpyxl
         ↓
رابط التنزيل يظهر في المحادثة مباشرة
```

### ٧.٧ ما لا يمكن للذكاء الاصطناعي فعله بشكل مباشر

| القيد | الحل المقترح في التطبيق |
|---|---|
| لا يملك واجهة بصرية | Frontend مخصص يعرض النتائج بشكل منسق |
| لا يتذكر بين الجلسات | حفظ المحادثة كاملة في PostgreSQL |
| لا يرى الصور داخل PDF | استخدام OCR أو استخراج النص فقط في المرحلة الأولى |

## ٨. القرارات المحسومة والنقاط المفتوحة

### ✅ جميع القرارات محسومة

| القرار | الخيار المُعتمد |
|---|---|
| حفظ الملف | يُحفَظ كاملاً — أكبر حجم ممكن بلا قيود |
| المستخدمون | متعددون — مدير وموظفون |
| صلاحية الموظف | كل الصلاحيات التشغيلية ما عدا صفحة الإعدادات |
| الهوية البصرية | بنفسجي حديث — وضع ليلي ونهاري — خط Cairo |
| نظام التنقل | Sidebar للحاسب + Bottom Nav للجوال |
| فقاعات المحادثة | كلاسيكية مع Streaming وأيقونات حالة الذكاء الاصطناعي |
| لوحة الملفات | يسار قابلة للطي |
| لوحة التحكم | شبكة Grid — 3 بطاقات حاسب / 2 تابلت / 1 جوال |
| زر الإرسال | أيقونة فقط — Enter للإرسال — Shift+Enter لسطر جديد |
| الشعار | أيقونة + نص "DataChat" — أيقونة منفردة في الأماكن الضيقة |
| الحالات الفارغة | رسالة واضحة + توجيه عملي + زر للخطوة التالية في كل حالة |
| الملف الشخصي | المدير فقط — تغيير الاسم والبريد وكلمة المرور وحذف الحساب |
| إحصائيات المدير | 4 أرقام: مستخدمون نشطون، مشاريع، ملفات مرفوعة، ملفات مُولَّدة، رسائل |
| اللغة | ثنائية اللغة — عربي / إنجليزي — زر تبديل في الشريط العلوي |
| الإشعارات | داخل التطبيق: اكتمال الملف، فشل الرفع، تسجيل دخول جديد (للمدير) |
| إدارة الجلسة | خروج عند إغلاق المتصفح — خمول 15 دقيقة — خيار تذكّرني |
| الإرشاد | جولة توجيهية 3-4 خطوات للموظف الجديد عند أول تسجيل دخول |
| إدارة المستخدمين | طريقتان: إنشاء مباشر أو رابط دعوة بالبريد (صالح 48 ساعة) |
| ملفات المشروع | متعددة (حتى 10) — الذكاء الاصطناعي يجيب عن الكل أو ملف محدد |
| تسمية المشروع | المستخدم يكتب الاسم يدوياً — مطلوب — قابل للتعديل لاحقاً |
| معاينة الملف | أول 5 صفوف (جداول) أو أول فقرة (PDF/Word) بعد الرفع مباشرةً |
| تصدير المحادثة | PDF منسّق أو نص عادي .txt — من قائمة خيارات المشروع |
| حجم الملف | 50 MB حد أقصى — مؤشر تقدم أثناء الرفع — رسالة واضحة عند التجاوز |
| نسخ الرسائل | زر 📋 على كل رسالة ذكاء اصطناعي — يتحول ✅ تأكيداً |
| تعديل الرسائل | يُعاد الإرسال تلقائياً — الردود اللاحقة تُحذف بعد تأكيد |
| تقييم الردود | 👍 / 👎 — السلبي يفتح حقل تعليق اختياري — يُحفَظ للمدير |
| تثبيت المشاريع | 📌 في زاوية البطاقة — قسم مثبّتة فوق باقي المشاريع |
| Markdown | كامل: عناوين، قوائم، غامق، جداول تفاعلية، كود |
| انقطاع الإنترنت | شريط تنبيه + رسالة خطأ + زر "أعد المحاولة" — الرسالة تبقى |
| تصميم PDF | احترافي بشعار ورأس وتذييل وجداول منسقة |
| الأوامر السريعة | محذوفة — محادثة حرة فقط |
| الجداول في المحادثة | تفاعلية كاملة (ترتيب، تصفية، بحث، تنزيل) |
| التحليل | عند الطلب فقط، لا يوجد تحليل تلقائي |
