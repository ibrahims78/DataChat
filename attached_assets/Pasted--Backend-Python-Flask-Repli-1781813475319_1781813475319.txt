Project Title: Comprehensive Sales & Inventory Management System (Syria - Multi-currency & Multilingual)

Objective: Develop a full-stack web application on Replit for managing sales, purchases, inventory, customers, and suppliers, mirroring the functionality of the provided Excel sheets. The application must support Syrian Pound (SYP) and US Dollar (USD) currencies, be fully bilingual (Arabic/English) with RTL support for Arabic, and feature a professional, modern UI with a unique visual identity.

Technology Stack:

Backend: Python 3.10+ (Flask, SQLAlchemy, Flask-Babel, Flask-Login, Flask-CORS)
Database: SQLite (managed by SQLAlchemy)
Frontend: React 18+ (Vite, React Router DOM, react-i18next)
Styling: Tailwind CSS (with tailwindcss-rtl plugin for RTL support)

Key Functionality & Logic:

CRUD Operations: Implement Create, Read, Update, Delete for all core models (Customers, Items, Warehouses, Suppliers, Sales, Purchases).
Inventory Management:
When a Sales transaction is completed, decrement Items.quantity for each Sales_Item.
When a Purchases transaction is completed (status 'تم الاستلام'), increment Items.quantity for each Purchase_Item.
Alerts/Notifications for Items.quantity < Items.reorder_level.
Customer Balance:
When a Sales transaction is completed, add Sales.total_amount to Customers.balance.
(Future: Implement payment recording to reduce Customers.balance).
Multi-currency Handling:
Allow users to select SYP or USD for Items prices, Sales total, and Purchases total.
Display currency symbols appropriately (ل.س / $).
No real-time currency conversion is required, simply record and display in the chosen currency for the transaction/item.
Search & Filtering: Implement search and basic filtering capabilities for lists of Customers, Items, Sales, Purchases.
Reporting (Basic):
Inventory Status (items below reorder level).
Sales Summary (by customer, by item, by date range).
Purchase Summary (by supplier, by item, by date range).
User Authentication & Authorization: Simple login/logout. (Roles can be used for basic access control, e.g., sales can only manage sales, warehouse can only manage inventory).
Frontend (React.js) Requirements:

Modular Design: Create separate components for each module (e.g., CustomerList, CustomerForm, ItemList, ItemForm, SaleForm, PurchaseForm, etc.).
Routing: Use react-router-dom for navigation between modules.
Bilingual Support:
Implement react-i18next for all text content (labels, buttons, messages).
Provide a language switcher (Arabic/English).
RTL/LTR: Dynamically apply dir="rtl" to the html tag when Arabic is selected, and dir="ltr" for English. Tailwind CSS with tailwindcss-rtl will handle most styling adjustments.
Professional & Modern UI:
Visual Identity: Clean, intuitive, responsive design. Use a consistent color palette (e.g., a primary brand color, secondary accent, neutral grays).
Layout: Responsive layout using Tailwind CSS grid/flexbox for optimal viewing on various devices.
Forms: Well-structured forms with clear labels, validation, and user-friendly input fields (e.g., dropdowns for Settings values).
Tables: Clear, sortable, and paginated tables for displaying data.
Navigation: Intuitive sidebar/top-bar navigation.
Currency Display: Format currency values appropriately (e.g., "1,250.00 ل.س" or "$35.50").
Backend (Flask) Requirements:

RESTful API: Create API endpoints for all CRUD operations on the data models.
JSON Handling: All API requests/responses should be in JSON format.
Database Interaction: Use SQLAlchemy to interact with the SQLite database.
Business Logic: Implement the inventory and customer balance update logic within the API endpoints.
Error Handling: Implement proper error handling and return meaningful error messages (e.g., 400 Bad Request, 404 Not Found, 401 Unauthorized).
Internationalization: Use Flask-Babel for backend messages if any, and for setting the locale based on frontend preference.
CORS: Configure Flask-CORS to allow requests from the React frontend.
User Sessions: Implement Flask-Login for managing user sessions.
